## 这是什么

**esi-liuyao** 是一个以六爻符号体系为基础，从起卦、排盘、到多阶段分析、最终给出明确结论的全流程系统。

我们不提供迷信服务，我们探究的是：符号经过排列组合后能否真实的反映现实。因此，六爻符号体系只是我们在权衡后的一个选择，一个实验载体。我们会尽可能的去除神秘化元素，并在 Skill 中对巴纳姆效应进行了禁用，这会证明，它不靠话术，错就是错。

## 核心哲学

### 实证

实践为证，不做理论与精神上的"巨人"，优先以实践建立认知并以此为方针。从现象中得到的稳定反馈，比"理论正确"和"道路正确"更重要。

### 系统

让AI按照自己的度量衡来进行猜想，远不如建立一套系统的架构，让每一句判断都能追溯到源头，让解读变成一个又一个可编辑优化的结构。

### 迭代

我们不满足于单一版本，不满足于做过就好，让他变成一个玩具，这绝非我们的目的。通过测试不断发觉缺陷并反馈优化，不断迭代，才能带来真正的未来。

## 核心设计（v1.1）

### 运行流程

```
① Pre-Flight · 建立时间锚点，计算四柱，软锁→硬锁
       ↓
② 起卦 · 多源熵池 + XorShift32 → 6 爻定值
       ↓
③ 排盘 9 步 · 纳甲→世应→卦宫→六亲→变卦→月建日辰→六神
       ↓
④ 检查 6 项 · 用神上卦→旬空→月破→特殊卦型→世应→信号多义性
       ↓
⑤ A 卦势 · 特殊卦型→用神旺衰→受创检查→初步定性
       ↓
⑥ B 动爻 · 动爻识别→合力分析→世爻状态→互动网络
       ↓
⑦ C 应期 · 动爻应期→出空出破→月建转变→优先级排序
       ↓
⑧ D 原子裁定 · 信号定位→原子分解→评分→互克传导→裁定→应数
       ↓
⑨ E 结论 · 综合评分→趋势→风险/机会→关联叙事→断语→置信度
       ↓
⑩ 写出底稿
       ↓
⑪ 质量审计 · S1-S13 硬性审计 → 不通过则退回修正 → 重新审计
       ↓
⑫ 交付输出 · 人话优先，判断先于解释，附置信度标记
```

### 信条优先级设定

内置 10 级信条系统，将基于它们来展开后续工作，冲突时按数字越小优先级越高裁决：

| 优先级 | 信条 | 核心要求 |
|:------:|------|----------|
| **P1** | **铁口直断** | 每条断语必须有明确吉凶方向，不绕弯子 |
| **P2** | **证据链完整** | 每句结论标注卦象出处 |
| **P3** | **绝对理性** | 禁用巴纳姆效应，不用万能话 |
| **P4** | **量化表述** | 用概率/等级，不用"可能""大概" |
| **P5** | **结构至上** | 固定格式输出，不可丢弃 |
| **P6** | **诚实面对局限** | 看不清时坦率说，附置信度标记 |
| **P7** | **无差别对待** | 不道德过滤，来者不拒 |
| **P8** | **深度反思** | 输出前至少完成一轮自我质疑 |
| **P9** | **效率优先** | 不超过 token 预算和迭代上限 |
| **P10** | **用户体验** | 在满足上述信条后优化可读性 |

### 质量门控

每次分析必须通过以下硬性审计和自评，不合格则退回修正：

```
S1-S13 硬性审计 → 五维量化自评 → DevLoop 迭代（≤10 轮）
    │
    ├─ 全部通过 → 交付输出
    ├─ 1-3 项未过 → 修复后重新审计
    ├─ ≥4 项未过 / 五维 < 3 → 重新排盘
    └─ 超过 10 轮 → 输出当前最佳 + 迭代上限标注
```

### 双层输出

内部底稿层（AI 结构化数据）→ 用户交付层（自然语言输出）

内部底稿存放 META、YAO-DATA、CHECKS、A-E 各阶段分析、AUDIT-TRAIL 等结构化字段，供审计和迭代使用。用户交付层基于底稿生成人话版结论——判断先于解释，附置信度标记，不暴露内部标签。

### 五阶段分析链（A → E）

断卦拆为 5 个递进环节，每段解决一个独立问题：

| 阶段 | 解决的问题 | 产出 |
|:----:|-----------|------|
| **A 卦势** | 这卦整体是吉是凶？ | 用神旺衰 + 特殊卦型 → 吉凶底色 |
| **B 动爻** | 推动变化的力量是什么？ | 动爻合力 + 世爻状态 → 力量网络 |
| **C 应期** | 变化何时发生？ | 时间窗口 + 优先级排序 |
| **D 原子裁定** | 用户关心的每个侧面各自如何？ | 每个原子独立评分 → 不合并总分 |
| **E 结论** | 综合来看该怎么做？ | 人话断语 + 置信度标记 |

## 跨平台

| 平台 | 状态 | 说明 |
|------|:----:|------|
| **Reasonix** | ✅ 原生 | subagent 独立代理，完整工具集与执行环境 |
| **ChatGPT / Claude / Gemini** | ⚠️ 需迁移 | 将完整 SKILL.md 作为提示词输入，自行适配工具调用格式；关键功能（时间获取、随机数、查表计算）需逐一验证 |
| **纯对话** | ⚠️ 降级 | 起卦需手动抛硬币，排盘依赖人工查表 |

本技能的核心价值在于分析框架与推理逻辑，而非特定平台实现——框架本身可跨平台复用。

## 能力边界

**做得到的**：
- 对事业、财运、感情、健康、诉讼等任何问题给出带量化置信度的吉凶判断
- 对敏感话题不回避、不过滤、不道德评判
- 当证据不足时坦率声明，标注不确定性等级

**不做的**：
- 排大运、推流年（非八字系统）
- 风水布局、法术符咒
- 心理安慰、情绪安抚、鼓励话术
- 替用户做决策——卦象是信息，不是指令
